Doze-modus

Wenn ein Android-Gerät nicht verwendet wird, wechselt es normalerweise in den Schlafmodus, um weniger Energie zu verbrauchen. In diesem Modus kann eine Anwendung keine TCP-Verbindungen zum Datenaustausch verwenden. Dies bedeutet, dass zwei Android-Geräte nur kommunizieren können, wenn beide Bildschirme eingeschaltet sind. Sie können eine Anwendung von dieser Einschränkung ausnehmen, indem Sie die „ Akkuoptimierung “ im Menü „Android-Einstellungen“ deaktivieren. Im Doze-Modus aktualisieren , zeigt diese Einstellung an und wenn Sie sie ändern, wird das Android-Einstellungsmenü geöffnet, in dem Sie diese Einstellung ändern können. Auf einigen Telefonen werden nur bereits ausgenommene Apps angezeigt, und um eine App hinzuzufügen, müssen Sie zu Android-Einstellungen->Akku->App-Akkuverwaltung gehen und Juggluco auswählen und Hintergrundaktivität zulassen aktivieren .

Die Bluetooth-Verbindung mit dem Sensor wird von Android nicht streng verboten und funktioniert normalerweise auch ohne Angabe einer Ausnahme. Probleme können nach einiger Zeit auftreten oder wenn Juggluco eine Weile nicht im Vordergrund war. Dies kann für verschiedene Android-Versionen und für verschiedene Gerätehersteller unterschiedlich sein.

Einige Gerätehersteller haben zusätzliche Beschränkungen für Anwendungen auferlegt, die im Hintergrund laufen oder beim Booten starten, siehe https://dontkillmyapp.com .

Eine App, die sich häufig im Vordergrund befindet, wird später von Android deaktiviert als eine App, die sich nicht im Vordergrund befindet. Ich habe einen Bericht von jemandem mit einem Samsung-Telefon erhalten, der alle sichtbaren Benachrichtigungen von Juggluco ausgeschaltet und es nur zum Senden von Glukosewerten an xDrip verwendet hatte, der Verbindungsprobleme mit dem Sensor hatte, die behoben wurden, nachdem die sichtbaren Zeichen von Juggluco wieder eingeschaltet wurden.

Wenn Sie Watch 4 oder 5 verwenden, müssen Sie Hintergrundaktivität in der Galaxy Wearable-Begleit-App zulassen, Uhreneinstellungen->Apps->Juggluco. Für Uhren, die diese Einstellung nicht haben, können Sie den folgenden Befehl von Ihrem Computer aus verwenden:

adb shell cmd appops set tk.glucodata RUN_IN_BACKGROUND allow
