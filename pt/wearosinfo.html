Wear OS

Uma versão especial do Juggluco pode ser executada diretamente em um relógio Wear OS. Ele precisa de um smartphone Android com NFC para digitalizar o sensor e, por meio de uma conexão TCP, esses dados são enviados para o relógio Wear OS. A partir daqui existem duas possibilidades:

  1. O sensor fica conetado com o telemóvel e o telemóvel envia cada valor de glicose que recebe via Bluetooth para o relógio Wear OS via TCP;

  2. Também é possível conetar diretamente o sensor com o relógio Wear OS. Para isso, precisa de um relógio Wear OS com boa conectividade Bluetooth.

Pode obter o Juggluco para Wear OS no Google Play no seu relógio. Algumas fotos são mostradas em https://www.juggluco.nl/JugglucoWearOS

Para que funcione faça o seguinte:

  • Certifique-se de que o Bluetooth e o WIFI estejam funcionando no relógio e no telemóvel;

  • Veja se o relógio tem espaço de armazenamento livre suficiente e RAM restante. É possível que o Juggluco precise ser reinstalado se o dispositivo ficar sem espaço de armazenamento em um momento crítico;

  • Faça a versão para telemóvel do Juggluco funcionar com o sensor. Deve escanear o sensor algumas vezes com o "Sensor via Bluetooth" definido. Não é necessário receber valores de glicose no telemóvel via Bluetooth;

  • Instale a versão Wear OS do Juggluco no seu relógio Wear OS e inicie-o;

  • No telemóvel, vá para o menu esquerdo->Relógio->Wear OS Configuração. Seu relógio deve ser exibido no controlo giratório. Pressione "Iniciar aplicação de monitorização";

  • Se tudo correu bem, em ambos os Juggluco é configurada uma conexão TCP e todos os dados do telemóvel são enviados para o relógio. Isso pode levar muito tempo se estiver usando Juggluco por muito tempo. Pode verificar se a conexão TCP está configurada na caixa de diálogo Espelhar (telemóvel: menu esquerdo do meio->Espelhar, observe: menu esquerdo->Espelhar). Uma conexão com o mesmo ID mostrado no controlo giratório do seu relógio deve ser mostrada. No relógio também deve conter o IP do telemóvel. Se não estiverem lá ou o lado do relógio não contiver um IP, pressione novamente em "Iniciar aplicação de monitorização". Se ainda não estiver lá, verifique a conexão Bluetooth e o aplicativo de telemóvel responsável pela conexão com seu relógio. (Quando o relógio já recebeu dados, pressionar "Iniciar aplicação de monitorização" tem o efeito de reenviar todos, então também pode reinstalar a versão de relógio do Juggluco. Mais tarde, se o IP correto não for mostrado, deve desligar e ligar a conexão externa (WIFI ou dados móveis) no telemóvel.

  • Se tudo correr bem, os mesmos dados exibidos na versão para telemóvel do Juggluco são exibidos na versão para relógio do Juggluco. Cada novo valor de glicose recebido pelo telemóvel é enviado via TCP para o relógio. Algumas configurações, como unidade, níveis de alarme de glicose e lembretes, também são transmitidas ao relógio. Alterações posteriores devem ser feitas no próprio relógio.

  • Se possui um relógio com boa conectividade Bluetooth e gosta de receber os valores de glicose no relógio sem precisar carregar o telemóvel, pode conectar a versão do relógio do Juggluco diretamente ao sensor. Depois que todos os valores forem enviados para o relógio, pode verificar "Conexão direta sensor-relógio" . Com isso, o "sensor via Bluetooth" é desativado no telemóvel e ativado no relógio. Os valores do stream agora são enviados do relógio para o telemóvel, como você pode verificar na caixa de diálogo Espelhar. Não pode alterar a "Conexão direta do sensor com o relógio" quando o relógio e o telemóvel não estiverem sincronizados.

  • Se deseja enviar valores do relógio para o telemóvel, você mesmo deve inverter a direção de envio na caixa de diálogo Espelhar, desmarcando-a no telemóvel e marcando-a no relógio. No telemóvel, " Receber de " também deve ser definido. Se tudo não estiver definido exatamente certo, não funciona. "Não altere valores" deve ser alterado. As etiquetas de valores são sempre enviadas com os valores. Se alterar a direção da transmissão de valores do relógio para o telemóvel, não poderá alterar as etiquetas. Não deve ser muito fácil;

Se o sensor tiver uma conexão Bluetooth direta com o relógio e seus valores forem enviados para um telemóvel, os valores digitalizados também devem ser enviados do telemóvel para o relógio e os valores do stream devem ser sincronizados antes da digitalização do sensor com o Juggluco (as configurações são definidas automaticamente desta forma.) Caso contrário, os valores antigos podem ser enviados de volta para o relógio ou as informações de autenticação modificadas não são enviadas para o relógio, o que resultará em erros de chave de desbloqueio .

Alterações na conexão também devem ser feitas apenas quando os dados estiverem sincronizados; os dados no telemóvel e no relógio devem ser cópias completas (com a exceção de que você pode deixar de fora todos os dados de quantidade).

Na primeira vez, quando muitos dados precisam ser transferidos do smartphone para o relógio, pode conectar o relógio ao carregador. Meu relógio liga o Wi-Fi quando está conectado ao carregador.

Conexão do relógio-telemóvel

Na inicialização, quando todos os dados são enviados do telemóvel para o relógio, precisa de uma conexão Wi-Fi entre o relógio e o telemóvel. Os dados posteriores são enviados em pequenas parcelas via TCP. No meu relógio que é redirecionado por Bluetooth, mas apenas quando o telemóvel possui uma conexão externa, por exemplo, Wi-Fi para sua rede doméstica, uma conexão de dados móveis ou um ponto de acesso portátil. Caso contrário, a conexão entre o telemóvel e o relógio é interrompida após algum tempo, porque o Wi-Fi é desligado pelo sistema para economizar bateria.

O próprio Juggluco deteta os IPs do outro lado, então não precisa configurá-los. Se alternar de Wi-Fi para dados móveis, os IPs são alterados automaticamente. Às vezes, precisa desligar e ligar o Wi-Fi ou os dados móveis em seu smartphone antes de funcionar (Juggluco determina o IP no momento em que o Wi-Fi ou os dados móveis são ativados, mas às vezes o IP ainda não está disponível).

A determinação automática do IP do outro lado é específica para a conexão com um relógio WearOS; para outras conexões Espelhar você ainda precisa especificar os IPs do outro lado.

mostrador do relógio

O Juggluco para Wear OS inclui um mostrador de relógio que mostra seu valor atual de glicose, além da hora e até quatro complicações. Para usá-lo, deve selecionar este mostrador do relógio, por exemplo, no aplicativo complementar do relógio em "baixado". Normalmente, você verá o último valor de glicose quando a tela for ativada. Quando o ecrã está no modo ambiente (sempre ligado), o valor da glicose só é atualizado quando o indicador dos minutos aumenta ou você pressiona um botão ou levanta o braço. Portanto, o valor exibido pode ser atrasado em até um minuto.

Também pode usar outro mostrador de relógio (por exemplo https://play.google.com/store/apps/details?id=com.ustwo.watchfaces.bits ) e adicionar um atalho de aplicativo ao Juggluco. Dessa forma, você pode saber seu valor de glicose pressionando esse atalho.

Com alguma programação, você também pode criar um mostrador de relógio separado que exibe o valor de glicose da transmissão de glicose de Juggluco.
