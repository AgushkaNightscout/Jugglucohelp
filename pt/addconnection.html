Adicionar conexão

Especifique uma conexão com outra instância de Juggluco. Juggluco em um dispositivo envia via IP/TCP dados para um Juggluco em outro dispositivo. Para estabelecer uma conexão, um dos Juggluco deve fazer contato numa porta e o outro deve fazer contato com essa porta. A porta em que esta instância do Juggluco está escutando foi fornecida na tela anterior (menu do meio esquerdo-> Espelho). Aqui especifica a porta para a qual esta instância de Juggluco contata se ela fizer contato ativamente. Se esta instância do Juggluco não puder contatar numa porta, selecione " Somente ativo ". Se o outro lado não puder contatar numa porta, selecione " Apenas passivo ", caso contrário, deixe " Ambos " marcado.e Ativo somente se selecionar aqui Somente passivo e Ambos se selecionar aqui Ambos .

Se esta instância do Juggluco entrar em contato ativamente, deve especificar o(s) IP(s) com o qual ela deve entrar em contato. Se esse host tiver vários IPs, especifique mais de um; por exemplo, quando dois dispositivos às vezes são conectados por meio de uma rede doméstica e, às vezes, diretamente como ponto de acesso portátil. Nunca especifique os IPs de vários dispositivos, nesse caso cada um receberá apenas parte dos dados.

Em todos os casos, exceto 'Somente ativo', pode selecionar ' Detectar '; neste caso Juggluco salva o primeiro IP contatando Juggluco como o IP do outro lado. Verificar se o dispositivo certo está fazendo contato pode ser feito de duas maneiras: Selecione ' Testar IP ' para testar o IP. Se selecionar Rótulo de teste , um contato só será estabelecido se ambos os lados tiverem especificado o mesmo rótulo.

Selecione receber de se quiser que este aplicativo funcione como um aplicativo espelho.

Se deseja que este dispositivo seja o remetente de dados, deve especificar quais dados enviar.

Quantidades : dados de dose, alimentos e desportos que especificou.

Scan: dados recebidos pela varredura do sensor (scan e histórico).

Stream : os dados recebidos por Bluetooth.

Às vezes, alguns dados já estão presentes no dispositivo receptor.  pode especificar até quando esses dados estão presentes:

Iniciar: sem dados presentes

Agora: todos os dados presentes

Ou uma data específica determinada pela posição inicial da tela.

Quando uma fonte de dados é adicionada a uma conexão existente, a data de início determina apenas a fonte de dados adicionada. Por exemplo, se Scans e Streams já foram enviados para o hospedeiro e os valores agora são adicionados, a data de início determina apenas os valores enviados.

Caso dois dispositivos estejam conectados por meio de uma conexão insegura, os dados podem ser assinados e criptografados especificando a mesma senha (no máximo) de 16 caracteres em ambos os lados da conexão. Selecione Guardar para salvar o que especificou. Excluir excluirá esta especificação de conexão.

Na situação mais simples, todas as conexões pertencem à sua rede doméstica, portanto, pode usar IPs locais.  também pode se conectar pela internet via Wi-Fi, nesse caso deve consultar o manual do seu modem sobre como encaminhar uma porta externa para um IP e porta na sua rede doméstica.

Dois telemóveis conectados por meio de uma conexão de dados móveis pela Internet podem se conectar diretamente um ao outro, se um deles puder escutar em uma porta de rede. Se ambos não puderem (por exemplo, se nenhum tiver um IP individual), eles ainda poderão se comunicar por meio de um terceiro dispositivo. Uma possibilidade é executar o Juggluco em um dispositivo Android (no mínimo Android 4.4) em casa conectado à internet por meio de um modem. Outra possibilidade é executar um programa de linha de comando pertencente ao Juggluco em um computador (por exemplo, seu PC ou Amazon AWS): http://jkaltes.byethost16.com/Juggluco/cmdline/index.html

É possível receber valores de glicose via Bluetooth em um dispositivo sem NFC (consulte http://jkaltes.byethost16.com/Juggluco/mirror): Inicialize o sensor via NFC no dispositivo NFC e, em seguida, envie os dados de digitalização e fluxo para o dispositivo não NFC. A seguir, inverta a conexão para a data do fluxo e desligue a conexão Bluetooth no dispositivo NFC e ligue-a no sem NFC dispositivo e também desligue o "sensor via Bluetooth" no dispositivo NFC e ligue-o no dispositivo não NFC. Assim, você pode fazer scan com um dispositivo e obter valores de glicose via Bluetooth em outro dispositivo. Mas em todos os dispositivos todos os dados do sensor devem estar disponíveis. Se os dispositivos trocarem dados, eles sempre devem receber dados de varredura e fluxo. Portanto, você deve sempre enviar os dados do Stream de volta para o dispositivo NFC e os dados de digitalização de volta para o dispositivo não NFC. Caso contrário, os dados ficarão fora de sincronia, fazendo com que dados de autenticação antigos sejam usados ​​ou novos dados sejam substituídos por dados antigos.

Os dados de Quantias são totalmente independentes.

problemas de rede

É necessária uma especificação complementar estrita em ambos os lados da conexão. Uma simples omissão pode impossibilitar a transmissão de dados.

É possível que a conexão pare e precise ser redefinida ligando ou desligando o WIFI ou pressionando Sincronizar. Além disso, alternar uma conexão ativa-somente-passiva-somente para ambos ou o contrário pode fazer a diferença.

Por exemplo, uma conexão relógio-telemóvel usando TCP sobre Bluetooth só funciona se o relógio estiver apenas ativo e o telemóvel apenas passivo. Sobre WIFI funciona em ambas as direções.

Você também deve considerar a mudança de IPs com mudança de conexão. Por exemplo, IPs diferentes devem ser usados ​​na rede doméstica e em um ponto de acesso portátil.
