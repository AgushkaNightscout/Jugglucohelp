Continue funcionando

Normalmente, quando um dispositivo Android não é usado, ele entra no modo suspensão para usar menos energia. Nesse modo, uma aplicação não pode usar conexões TCP para trocar dados. Isso significa que dois dispositivos Android só podem se comunicar quando ambas as telas estiverem ligadas.  pode isentar uma aplicação dessa restrição desativando a ' Otimização da bateria ' no menu de configurações do Android. Atualizar no modo suspensão, mostra essa configuração e alterá-la abre o menu de configuração do Android, onde pode alterar essa configuração. Em alguns telemóveis, apenas as aplicações já isentas são mostradas e, para adicionar uma aplicação, deve ir para configurações do Android->bateria->gerenciamento de bateria da aplicação, selecionar Juggluco e ativar permitir atividade em segundo plano .

A conexão Bluetooth com o sensor não é estritamente proibida pelo Android e normalmente também funcionará sem especificar uma isenção. Mas isso pode ser diferente para diferentes versões do Android e para diferentes fabricantes de dispositivos. Para ter certeza ou depois que a aplicação perder alguns valores de glicose no final da noite, pode desativar a economia de bateria para esta aplicação, mesmo que não use o TCP para a função de espelhar.

Alguns fabricantes de dispositivos impuseram restrições adicionais às aplicações executados em segundo plano ou abertos na inicialização, consulte https://dontkillmyapp.com .

Uma aplicação frequentemente em primeiro plano é desativada pelo Android mais tarde do que uma aplicação que não está em primeiro plano. Recebi um relato de uma pessoa com um telemóvel Samsung que havia desligado todas as notificações visíveis do Juggluco, usando apenas para enviar valores de glicose para o xDrip, que estava com problemas de conexão com o sensor que foram resolvidos após ligar novamente os sinais visíveis do Juggluco.
