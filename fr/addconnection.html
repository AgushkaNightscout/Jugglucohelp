Ajouter une connexion

Spécifiez une connexion avec une autre instance de Juggluco. Juggluco sur un appareil envoie via TCP/IP des données à un Juggluco sur un autre appareil. Pour établir une connexion, l'un des Juggluco doit écouter sur un port et l'autre doit établir un contact avec ce port. Le port sur lequel cette instance de Juggluco écoute a été indiqué dans l'écran précédent (menu central gauche -> Miroir). Ici, vous spécifiez le port que cette instance de Juggluco contacte si elle établit activement un contact. Si cette instance de Juggluco ne peut pas écouter sur un port, sélectionnez " Actif uniquement ". Si l'autre côté ne peut pas écouter sur un port, sélectionnez " Passif uniquement ", sinon laissez " Les deux " coché, et Actif uniquement si vous sélectionnez ici Passif uniquement et Les deux si vous sélectionnez ici Les deux .


Si cette instance de Juggluco entre activement en contact, vous devez spécifier la ou les adresses IP avec lesquelles elle doit entrer en contact. Si cet hôte a plusieurs adresses IP, vous en spécifiez plusieurs ; par exemple lorsque deux appareils sont parfois connectés via un réseau domestique et parfois directement en tant que hotspot portable. Ne spécifiez jamais les adresses IP de plusieurs appareils, dans ce cas, ils ne recevront chacun qu'une partie des données.


Dans tous les cas sauf 'Actif uniquement ', vous pouvez sélectionner ' Détecter ' ; dans ce cas, Juggluco enregistre la première adresse IP contactant Juggluco comme adresse IP de l'autre côté. Vérifier si le bon appareil établit le contact peut se faire de deux manières : Sélectionnez « Test IP » pour tester l'IP. Si vous sélectionnez Tester l'étiquette , un contact n'est établi que si les deux parties ont spécifié la même étiquette.


Sélectionnez recevoir de si vous souhaitez que cette application fonctionne comme une application miroir. Si vous souhaitez que cet appareil soit l' expéditeur des données, vous devez spécifier les données à envoyer :

Quantités : données de dose, d'alimentation et de sport que vous avez spécifiées.

Scans : données reçues en scannant le capteur (scans et historique).

Stream : les données reçues par Bluetooth.


Parfois, certaines données sont déjà présentes sur l'appareil récepteur. Vous pouvez spécifier jusqu'à quand ces données sont présentes :

Début : aucune donnée présente

Maintenant : toutes les données sont présentes

Ou une date spécifique déterminée par la position de début de l'écran.


Lorsqu'une source de données est ajoutée à une connexion existante, la date de début détermine uniquement la source de données ajoutée. Par exemple, si des analyses et des flux ont déjà été envoyés à l'hôte et que des quantités sont maintenant ajoutées, la date de début détermine uniquement les quantités envoyées.


Dans le cas où deux appareils sont connectés via une connexion non sécurisée, les données peuvent être signées et cryptées en spécifiant le même mot de passe (maximum) de 16 caractères des deux côtés de la connexion. Sélectionnez Enregistrer pour enregistrer ce que vous avez spécifié. Supprimer supprimera cette spécification de connexion.


Dans la situation la plus simple, toutes les connexions appartiennent à votre réseau domestique, vous pouvez donc utiliser des adresses IP locales. Vous pouvez également vous connecter sur Internet via Wi-Fi, dans ce cas, vous devriez consulter le manuel de votre modem pour savoir comment transférer un port externe vers une adresse IP et un port sur votre réseau domestique.


Deux smartphones connectés via une connexion de données mobile sur Internet peuvent se connecter directement l'un à l'autre, si l'un d'eux peut écouter sur un port réseau. Si les deux ne peuvent pas (par exemple si aucun n'a d'adresse IP individuelle), ils peuvent toujours communiquer entre eux via un troisième appareil. Une possibilité est d'exécuter Juggluco sur un appareil Android (minimum Android 4.4) à la maison connecté à Internet via un modem. Une autre possibilité consiste à exécuter un programme en ligne de commande appartenant à Juggluco sur un ordinateur (par exemple, votre PC ou Amazon AWS) : https://www.juggluco.nl/Juggluco/cmdline/index.html

Il est possible de recevoir des valeurs de glucose via Bluetooth sur un appareil sans NFC (voir https://www.juggluco.nl/Juggluco/mirror): Initialisez le capteur via NFC sur l'appareil NFC, puis envoyez les données de numérisation et de diffusion à l'appareil non NFC. Ensuite, inversez la connexion pour la date de diffusion et désactivez la connexion Bluetooth dans l'appareil NFC et activez-la dans le périphérique non-NFC et désactivez également le "capteur via Bluetooth" dans le périphérique NFC et activez-le dans le périphérique non NFC. Ainsi, vous pouvez numériser avec un appareil et obtenir des valeurs de glucose via Bluetooth sur un autre appareil. Mais sur tous les appareils, toutes les données des capteurs doivent être disponibles. Si les appareils échangent des données, ils doivent toujours recevoir à la fois les données de balayage et de flux. Vous devez donc toujours renvoyer les données de flux vers l'appareil NFC et les données de numérisation vers l'appareil non NFC. Sinon, les données seront désynchronisées, ce qui entraînera l'utilisation d'anciennes données d'authentification ou l'écrasement de nouvelles données par d'anciennes données.

Les données Quantités sont totalement indépendantes.

Problèmes de réseau

Une spécification complémentaire stricte des deux côtés de la connexion est nécessaire. Une simple omission peut rendre impossible la transmission des données.

Il est possible que la connexion se bloque et doive être réinitialisée en activant ou désactivant le WIFI ou en appuyant sur Sync. De plus, basculer une connexion active uniquement / passive uniquement vers les deux ou l'inverse peut faire une différence.

Par exemple, une connexion montre-téléphone utilisant TCP sur Bluetooth ne fonctionne que si la montre est active uniquement et le téléphone passif uniquement. Sur WIFI, cela fonctionne dans les deux sens.

Vous devez également envisager le changement d'adresses IP avec le changement de connexion. Par exemple, différentes adresses IP doivent être utilisées sur le réseau domestique plutôt que sur un hotspot portable.
