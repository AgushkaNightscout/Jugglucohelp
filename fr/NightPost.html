Téléchargez sur un serveur Nightscout.

Dans la plupart des cas, vous pouvez également utiliser le serveur Web dans Juggluco ou juggluco-server , seuls AAPS et NSclient refusent de fonctionner lorsqu'ils ne sont pas connectés à cgm-remote-monitor . Le serveur Web de Juggluco donne également des montants sous forme de traitements Nightscout ( https://www.juggluco.nl/Juggluco/webserver.html ). Le téléchargeur ne télécharge que les valeurs de glucose.

Toutes les valeurs de flux sont téléchargées sur le serveur Nightscout au moment où elles arrivent. Lorsque plusieurs capteurs sont utilisés, toutes les valeurs de tous les capteurs sont téléchargées. Certaines applications/visages/widgets de montre Garmin supposent que les valeurs sont espacées de 5 minutes. Lorsqu'ils reçoivent des valeurs de glucose à une minute d'intervalle, ils n'affichent que le dernier cinquième de la courbe. Pour cette raison, le serveur Web intégré à Juggluco espace les valeurs de 5 minutes, sauf lorsque "interval=secs" est défini sur une valeur inférieure.

URL : l'url doit être sous la forme https://hostname:port par exemple https://myname.herokuapp.com:6789

Actif : Active le téléchargement

Renvoyer les données : téléchargez à nouveau toutes les données sur le serveur Nightscout

Envoyer maintenant : normalement, les données sont immédiatement envoyées lorsqu'elles arrivent du capteur ou d'une connexion miroir. Appuyez sur ce bouton lorsque, pour une raison quelconque, une tentative précédente a échoué.

La version WearOS Juggluco 4.15.0 et les versions ultérieures peuvent également télécharger des valeurs de glucose sur Nightscout. Dans presque tous les cas il vaut mieux les envoyer via une version Juggluco sur téléphone ou tablette ou via serveur Juggluco sur ordinateur. Sur WearOS, cela videra la batterie en particulier lorsqu'elle ne peut pas contacter le serveur Nightscout. Il vaut donc mieux désactiver Actif sur WearOS lorsqu'il n'y a pas de connexion fiable.
