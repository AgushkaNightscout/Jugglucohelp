Wear OS

Connexion directe capteur-montre : Ceci détermine si la montre doit être directement connectée au capteur. Cela ne peut être modifié que lorsque la montre et le téléphone sont synchronisés. Utilisez Miroir -> Sync et Réinit sur le téléphone et la montre pour synchroniser le téléphone et la montre.

Initialiser appli montre : Il faut utiliser cette fonction une fois après l'installation de Juggluco sur la montre WearOS et son démarrage.

Défauts : Cela définit les paramètres de connexion à ses valeurs par défaut. Il s'agit des paramètres qui peuvent être modifiés dans le menu de gauche->Miroir sur la montre et le menu central gauche->Miroir sur le téléphone. Il se compose de paramètres tels que "Actif uniquement" et "Scans". Ce bouton doit être utilisé lorsque les paramètres ont été modifiés accidentellement par l'eau de la douche ou pendant le sommeil. Si vous réinitialisez en réinstallant la version de la montre ou en effaçant les données, vous devez également le faire pour supprimer la connexion miroir sur le téléphone.

Une version spécifique de Juggluco est faite peut fonctionner directement sur une montre Wear Os. Toutefois Juggluco besoin d'être aussi installé sur un smartphone Android avec NFC pour scanner le capteur (initialisation) et via une connexion TCP ces données sont envoyées à la montre Wear OS. Ensuite vous avez deux possibilités :

  1. Le capteur reste connecté au smartphone et le smartphone envoie chaque valeur de glucose qu'il reçoit via Bluetooth à la montre Wear OS via TCP ;

  2. Il est également possible de connecter directement le capteur à la montre Wear OS. Pour cela, vous avez besoin d'une montre Wear OS avec une bonne connectivité Bluetooth.

Vous pouvez obtenir Juggluco pour Wear OS via le lien sur le site de Juggloco. Certaines images sont présentées sur https://www.juggluco.nl/JugglucoWearOS

Pour le faire fonctionner, procédez comme suit :

  • Assurez-vous que Bluetooth et WIFI fonctionnent sur la montre et le téléphone ;

  • Assurez-vous que la montre dispose de suffisamment d'espace de stockage libre et de RAM. Il est possible que Juggluco doive être réinstallé si l'appareil manque d'espace de stockage à un moment critique ;

  • Faites en sorte que la version téléphone de Juggluco fonctionne avec le capteur. Vous devez scanner le capteur plusieurs fois avec le réglage "Capteur via Bluetooth". La réception des valeurs de glucose sur le téléphone via Bluetooth n'est pas nécessaire ;

  • Installez la version Wear OS de Juggluco sur votre montre Wear OS et démarrez-la ;

  • Sur le téléphone, allez dans le menu de gauche-> Montre-> Wear OS -> Config. Votre montre doit être affichée dans le menu déroulant. Appuyez sur "Initialiser appli montre" ;

  • Si tout s'est bien passé, une connexion TCP est configurée dans les deux Juggluco et toutes les données du téléphone sont envoyées à la montre. Cela peut prendre beaucoup de temps si vous avez utilisé Juggluco pendant longtemps. Vous pouvez vérifier si la connexion TCP est configurée en regardant dans la boîte de dialogue Miroir (téléphone : menu central gauche->Miroir, montre : menu gauche->Miroir). Une connexion avec le même identifiant que celui indiqué dans le menu déroulant pour votre montre doit être affichée. Sur la montre, il doit également y avoir l'adresse IP du téléphone. S'il n'y en a pas ou si le côté montre ne contient pas d'adresse IP, appuyez à nouveau sur "Initialiser appli montre". Si l’IP n'y est toujours pas, vérifiez la connexion Bluetooth et l'application du téléphone responsable de la connexion à votre montre (lorsque la montre a déjà reçu des données, appuyer sur "Init watch app" a pour effet de tout renvoyer, vous pouvez donc tout aussi bien réinstaller la version montre de Juggluco). Si cela ne fonctionne toujours pas et si la bonne adresse IP n'est pas affichée, vous devez activer et désactiver la connexion externe (WIFI ou données mobiles) sur le téléphone.

  • Si tout s'est bien passé les mêmes données, telles qu'affichées sur la version téléphone de Juggluco, sont affichées sur la version montre de Juggluco. Chaque nouvelle valeur de glucose reçue par le téléphone est envoyée via TCP à la montre. Certains paramètres tels que l'unité, les niveaux d'alarme de glucose et les rappels sont également transmis à la montre. Des modifications ultérieures doivent être apportées sur la montre elle-même.

  • Si vous avez une smartwatch avec une bonne connectivité Bluetooth et que vous préférez recevoir les valeurs de glucose sur votre montre sans avoir à transporter votre téléphone, vous pouvez connecter la version montre de Juggluco directement au capteur. Une fois toutes les valeurs envoyées à la montre, vous pouvez cocher "Connexion directe capteur-montre" . Ainsi, le "capteur via Bluetooth" est désactivé dans le téléphone et activé dans la montre. Les valeurs de flux sont désormais envoyées de la montre au téléphone, comme vous pouvez le vérifier dans la boîte de dialogue Miroir. Vous ne pouvez pas modifier la "Connexion directe capteur-montre" lorsque la montre et le téléphone ne sont pas synchronisés.

  • Si vous souhaitez envoyer des quantités de la montre au téléphone, vous devez inverser vous-même la direction d'envoi dans la boîte de dialogue Miroir en la décochant sur le téléphone et en la cochant sur la montre. Sur le téléphone " Recevoir de " doit également être défini. Si tout n'est pas réglé exactement, cela ne fonctionne pas. "Ne modifiez pas les quantités" doit être activé. Les étiquettes de quantités sont toujours envoyées avec les quantités. Si vous changez le sens de transmission des quantités de la montre au téléphone, vous ne pouvez pas changer les libellés. Cela ne devrait pas être trop difficile.

Si le capteur a une connexion Bluetooth directe avec la montre et que ses valeurs sont envoyées à un smartphone, les valeurs scannées doivent également être envoyées du téléphone à la montre et les valeurs de flux doivent être synchronisées avant de scanner le capteur avec Juggluco (les paramètres sont configurés automatiquement de cette façon). Sinon, les anciennes valeurs pourraient être renvoyées à la montre ou les informations d'authentification modifiées ne seraient pas envoyées à la montre, ce qui entraînerait des erreurs de clé de déverrouillage .

Les modifications de la connexion ne doivent également être effectuées que lorsque les données sont synchronisées ; les données sur le téléphone et la montre doivent être des copies complètes (à l'exception que vous pouvez omettre toutes les données de quantités).

La première fois, lorsque beaucoup de données doivent être transférées du smartphone vers la montre, vous pouvez connecter la montre au chargeur. Ma montre active le Wi-Fi lorsqu'elle est connectée au chargeur.

Connexion montre-téléphone

Lors de l'initialisation, lorsque toutes les données sont envoyées du téléphone à la montre, vous avez besoin d'une connexion Wi-Fi entre la montre et le téléphone. Les données ultérieures sont envoyées en petits morceaux via TCP. Sur ma montre qui est redirigé via Bluetooth, cela fonctionne uniquement lorsque le téléphone dispose d'une connexion externe par exemple Wi-Fi vers votre réseau domestique, une connexion de données mobiles ou un point d'accès portable. Sinon, la connexion entre le téléphone et la montre s'arrête après un certain temps, car le Wi-Fi est désactivé par le système pour économiser la batterie.

Juggluco détecte lui-même les adresses IP de l'autre côté, vous n'avez donc pas besoin de les définir. Si vous passez du Wi-Fi aux données mobiles, les IP sont automatiquement modifiées. Parfois, vous devez éteindre et rallumer le Wi-Fi ou les données mobiles sur votre smartphone avant qu'il ne fonctionne (Juggluco détermine l'IP au moment où le Wi-Fi ou les données mobiles sont activés, mais parfois l'IP n'est alors pas encore disponible).

La détermination automatique de l'adresse IP de l'autre côté est spécifique à la connexion avec une montre WearOS ; pour les autres connexions miroir, vous devez toujours spécifier les adresses IP de l'autre côté.

Cadran de la montre

Juggluco pour Wear OS comprend un cadran de montre qui affiche votre niveau de glucose actuel en plus de l'heure et jusqu'à quatre complications. Pour l'utiliser, vous devez sélectionner ce cadran de montre par exemple dans votre application compagnon de montre sous "téléchargé". Normalement, vous verrez la dernière valeur de glucose lorsque l'écran se réveillera. Lorsque l'affichage est en mode toujours allumé, la valeur de glucose n'est mise à jour que lorsque l'indicateur des minutes augmente ou que vous appuyez sur un bouton ou que vous levez le bras. Ainsi, la valeur affichée peut être retardée jusqu'à une minute.

Il existe également une complication qui utilise la diffusion Glucodata de la version WearOS pour afficher la valeur de glucose sur chaque cadran de montre qui accepte la complication : https://github.com/pachi81/GlucoDataHandler/releases
